Background

About Me

I'm a Senior Full-Stack Laravel Developer operating under the KodaKodra brand. I've been building web applications professionally since 2019 — but with the kind of hours and depth of focus that compress years into something more. This is what I do, and I do it seriously.

My stack is focused, not sprawling. Laravel is my primary framework — I know it deeply, not just the happy path. I design normalised database schemas, architect role-based permission systems, write clean Eloquent relationships, and build responsive Bootstrap front-ends that actually work on mobile. Every project follows the same disciplined workflow: migration → model → requests → controller → policy → routes → views. No shortcuts.

Security isn't a checklist I run through before shipping. It's a mindset baked into every design decision — CSRF protection, XSS prevention, mass-assignment protection, parameterised queries — all baseline, not optional extras. I think about how something can be abused before I write it, not after.

My biggest current project is a community social network built entirely in Laravel — featuring posts, communities, comments, profiles, privacy controls, a points and perks economy, leaderboards, and a shop. The entire platform is designed around one idea: incentivise good behaviour rather than punish bad behaviour. It's nearly complete, and it's the most technically complex thing I've built.

Outside of code, I hold a Level 3 qualification in Team Leading & Management, which isn't something I use to pad a CV — it reflects a genuine interest in how people and teams work well together. I'm as comfortable thinking about user motivation and community dynamics as I am about database indexes.

Philosophy

How I think about code

Code is communication
The next person to read your code knows nothing about what you were thinking. Write for them. Names, comments, structure — all of it is communication.
Simple beats clever
KISS isn't a beginner's principle. It's what experienced developers land on after trying to be too clever too many times. Readable, simple code is harder to write than complex code.
Security is design
If security is an afterthought, it's already too late. It needs to be part of the initial architecture decision, not bolted on at the end of a sprint.
UX is a technical concern
Good code that produces a confusing experience isn't good enough. The user's mental model matters as much as the data model.
